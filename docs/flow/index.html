架构风格

处理器(Handler)是整个框架的内核，它承载了一次请求处理的主要逻辑，与传统框架不同的是，FLuff 的内核是作为组件提供的。在开始一切之前，你首先要做的是选用一个 Handler 作为框架的核心。

Handler 的选择对于整体架构至关重要，它会影响到后续的应用开发方式，并且选择之后很难替换。Fluff 提供了五种不同的 Handler，它们与中间件配合，可以构建出不同的风格的架构。

我们在快速入门章节介绍了一种模式：单接口模式，通过 Handler 调用一个请求处理程序，并且不需要与任何中间件搭配。在接下来的几章，我们将介绍另外的五种模式。

  • 单接口注入模式
  • 路由模式
  • 直连路由模式
  • Ioc 路由模式
  • 混合路由模式
