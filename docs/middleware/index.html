中间件

有时你会想在 Application 调用 controller 之前或之后做一些事情，比如记录日志或开放 CORS 请求，这时你可以使用中间件 (middleware). 中间件是以 controller 为核心，层层嵌套的组件。中间件是栈结构的，最外层的中间件最先被调用。

*---------------------*
|     middleware2     |
|  *---------------*  |
|  |  middleware1  |  |
|  |   *-------*   |  |
|  |   |       |   |  |   中间件模型的结构
|  |   | core  |   |  |
|  |   |       |   |  |
|  |   *-------*   |  |
|  *---------------*  |
*---------------------*

Fluff 的的中间件遵循 PSR-15 标准，所以你的中间件必须实现 Psr\Http\Server\MiddlewareInterface 接口。

use Psr\Http\Message\ResponseInterface;
use Psr\Http\Message\ServerRequestInterface;
use Psr\Http\Server\MiddlewareInterface;
use Psr\Http\Server\RequestHandlerInterface;

class SayHelloMiddleware implements MiddlewareInterface
{
    public function process(ServerRequestInterface $request, RequestHandlerInterface $handler): ResponseInterface
    {
        $request = $request->withAttribute('say', 'good bye!');
        $response = $handler->handle($request);
        $response = $response->withStatus(404);
        return $response;
    }
}

以上案例中，我们定义了一个中间件，process 方法第二个参数是一个 request-handler 对象，request-handler 接受一个 server-side request 对象。当调用 RequestHandlerInterface::handle 时，会触发下一层中间件，最终会调用 controller，返回一个 PSR-7 response 对象。

我们可以方便的围绕 controller 做一些事情，也可以不执行 RequestHandlerInterface::handle 中断程序，并构建一个 response 对象直接返回，例如用户的权限检查未通过的场景等。

中间件只能在 route 中注册，当你使用 withRoute 方法时，在第四个参数中，将中间件名称添加到键名为 middlewares 的数组中，数组的键值越大，越靠近外层，越先被执行。

$app->withRoute('GET', '/user', 'userController', [
    'middlewares' => [
        SayHelloMiddleware::class,
        ...
    ]
])

添加全局中间件

用 Application::withMiddleware方法添加一个全局中间件，withMiddleware 方法接受一个中间件名称。

$app->withMiddleware('GlobalMiddleware');
