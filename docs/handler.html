处理器 (Handler)

处理器(Handler)是整个框架的内核，它承载了一次请求处理的主要逻辑，与传统框架不同的是，FLuff 的内核是作为组件提供的。在开始一切之前，你首先要做的是选用一个 Handler 作为框架的核心。

Handler 的选择对于整体架构至关重要，它会影响到后续的应用开发方式，并且选择之后很难替换。所以，了解每一种处理器的运行机制是必要的。本章将介绍三种子处理器

子处理器 (Sub Handler)

子处理器可以看做是对单个接口的处理逻辑，它没有路由的调度，只是对业务代码的简单的包装。实际上，我们在快速入门章节中已经使用到了一个子处理器 Handler. 下图展示了子处理器与整体结构的关系：

*------------*
| Middleware |      *-------------*
|  Builder   |<-----|   Router    |
*------------*      | Middleware  |
      |             *-------------*
      V                   ^
*----------*         [Route Data]
|  Router  |       [Sub Middlewares]
|  Handler |
*----------* 
      |
      V
*------------*
|    Sub     |
| Middleware |
|  Builder   |
*------------*
      |
      V
*----------*
|   Sub    |
|  Handler |
*----------*
